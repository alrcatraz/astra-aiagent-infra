## 触发条件

- VPS 升配后服务异常（Gateway 离线、Synapse 不可达）
- 磁盘分区操作被安全策略阻止（mkfs.xfs 等）
- 系统重启后关键文件丢失（/tmp 不持久）
- E2EE cross-signing 全清需要重建

## 诊断流程

### 第一步：确认 VPS 状态

```bash
ssh vps-hk "df -h; free -h; uptime; systemctl is-active matrix-synapse postgresql"
```

### 第二步：检查数据完整性

- Synapse 数据库是否可访问
- Hermes .env 配置是否正确
- Gateway 日志是否有 E2EE 错误

## 修复步骤

### [L3] VPS 升配恢复流程

1. **先备份再操作** — 参考 `pre-upgrade-server-backup`（跨机器备份）
2. **磁盘操作** — 用 `fdisk` 分区 + `mkfs.xfs`（注意：VPS 安全策略可能阻止，需用户手动执行）
3. **避免 /tmp** — 关键中间产物写入持久路径（如 /root/）
4. **E2EE 恢复** — 参考 `full-e2ee-recovery-after-server-rebuild`

### [L3] E2EE 恢复（关键！）

VPS 升配后最常见的问题是 E2EE 完全中断。完整流程见 `full-e2ee-recovery-after-server-rebuild`，关键点：

- 使用**全新 device ID**（带时间戳后缀）

## 已知陷阱

| 陷阱 | 说明 |
|:----|:------|
| mkfs.xfs 被安全限制阻止 | VPS 服务商可能禁止格式化操作，需用户手动执行 |
| /tmp 不持久 | 重启后丢失，关键文件写入 /root/ 或 /home/ |
| E2EE 恢复需用户配合 | Variant B 必须用户在 Element Web 设安全短语，不是纯自动化流程 |
| 跨机器备份 | 同一台机器的不同分区不是有效备份 |
